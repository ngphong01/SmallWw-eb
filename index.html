HTML Cơ Bản

HTML (HyperText Markup Language) là ngôn ngữ đánh dấu tiêu chuẩn để tạo các trang web. HTML mô tả cấu trúc của trang web bằng cách sử dụng các thẻ (tags).

Cấu trúc cơ bản của HTML


<!DOCTYPE html>
<html lang="vi">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Tiêu đề trang</title>
  </head>
  <body>
    <h1>Chào mừng đến với HTML</h1>
    <p>Đây là đoạn văn đầu tiên.</p>
  </body>
</html>

Các thẻ HTML cơ bản

Thẻ tiêu đề (Headings)

Tiêu đề H1 - Quan trọng nhất

Tiêu đề H2 - Phụ đề chính

Tiêu đề H4 - Phụ đề cấp 3

<h3>Tiêu đề H3 - Phụ đề cấp 2</h3>

<h5>Tiêu đề H5 - Phụ đề cấp 4</h5>

Thẻ văn bản

  • <p>: Đoạn văn bản
  • <strong>: Văn bản quan trọng (in đậm)
  • <em>: Văn bản nhấn mạnh (in nghiêng)
  • <b>: Văn bản in đậm (chỉ định dạng)
  • <i>: Văn bản in nghiêng (chỉ định dạng)
  • <u>: Văn bản gạch chân
  • <s>: Văn bản gạch ngang
  • <mark>: Văn bản đánh dấu
  • <small>: Văn bản nhỏ
  • <sub>: Chỉ số dưới (H2O)
  • <sup>: Chỉ số trên (E=mc2)

HTML Nâng Cao

Phần này sẽ giúp bạn hiểu sâu hơn về HTML5 và các kỹ thuật nâng cao để tạo ra các trang web hiện đại, semantic và accessible.

HTML5 Semantic Elements

HTML5 cung cấp các thẻ semantic giúp tạo cấu trúc rõ ràng và có ý nghĩa:

  • <header> - Đầu trang
  • <nav> - Menu điều hướng
  • <main> - Nội dung chính
    • <article> - Bài viết
    • <section> - Phần
  • <aside> - Sidebar
  • <footer> - Chân trang

<!DOCTYPE html>
<html lang="vi">
<head>
  <meta charset="UTF-8">
  <title>Trang Web Semantic</title>
</head>
<body>
  <header>
    <h1>Tên Website</h1>
    <nav>
      <ul>
        <li><a href="#home">Trang chủ</a></li>
        <li><a href="#about">Giới thiệu</a></li>
      </ul>
    </nav>
  </header>

  <main>
    <article>
      <header>
        <h2>Tiêu đề bài viết</h2>
        <time datetime="2025-01-15">15 tháng 1, 2025</time>
      </header>
      <p>Nội dung bài viết...</p>
    </article>
    
    <section>
      <h2>Phần liên quan</h2>
      <p>Nội dung phần...</p>
    </section>
  </main>

  <aside>
    <h3>Sidebar</h3>
    <p>Nội dung bên...</p>
  </aside>

  <footer>
    <p>© 2025 Tên Website</p>
  </footer>
</body>
</html>

CSS Cơ Bản

CSS (Cascading Style Sheets) là ngôn ngữ định dạng giao diện cho HTML. CSS kiểm soát cách các phần tử HTML được hiển thị trên màn hình, giấy, hoặc các phương tiện khác.

Các cách áp dụng CSS

1. Inline CSS

Đây là inline CSS - áp dụng trực tiếp trong thẻ HTML 


<p style="color: red; font-weight: bold;">Inline CSS</p>

2. Internal CSS


<head>
  <style>
    p {
      color: blue;
      font-size: 18px;
    }
  </style>
</head>

3. External CSS


<head>
  <link rel="stylesheet" href="styles.css">
</head>

/* styles.css */
p {
  color: green;
  font-size: 16px;
}

Pseudo-elements

Văn bản có pseudo-elements

Chữ cái đầu tiên của đoạn này sẽ được định dạng đặc biệt. Đây là cách tạo hiệu ứng drop cap trong typography.

Dòng đầu tiên của đoạn văn này sẽ có định dạng khác. Bạn có thể thấy sự khác biệt khi đọc. 


/* Pseudo-elements */
.pseudo-element-example::before {
  content: "👉 ";
  color: blue;
}
.pseudo-element-example::after {
  content: " ✨";
  color: gold;
}
.first-letter-example::first-letter {
  font-size: 2em;
  font-weight: bold;
  float: left;
}
.first-line-example::first-line {
  font-weight: bold;
  color: red;
}

CSS Box Model

Box Model là khái niệm cơ bản nhất trong CSS. Mọi phần tử HTML đều là một "hộp" gồm: content, padding, border, và margin.

CONTENT
200x100px
  • ■ Content (200x100)
  • ■ Border (5px)
  • ■ Padding (20px)
  • ■ Margin (30px)

/* Box Model */
.box-model-example {
  width: 200px;          /* Chiều rộng content */
  height: 100px;         /* Chiều cao content */
  padding: 20px;         /* Khoảng cách bên trong */
  border: 5px solid blue; /* Đường viền */
  margin: 30px;          /* Khoảng cách bên ngoài */
}

/* Box-sizing */
.box-border {
  box-sizing: border-box; /* Tính width/height bao gồm padding và border */
}

/* Shorthand properties */
margin: 10px;                    /* Tất cả 4 phía */
margin: 10px 20px;               /* Trên/dưới | Trái/phải */
margin: 10px 20px 15px;          /* Trên | Trái/phải | Dưới */
margin: 10px 20px 15px 25px;     /* Trên | Phải | Dưới | Trái */

Đoạn văn bình thường

Đoạn văn có thẻ strong - sẽ có border trái 


/* Attribute Selectors nâng cao */
input[type="text"]:not([disabled]) { border: 2px solid green; }
input[class*="btn"] { padding: 10px; }          /* class chứa "btn" */
input[class^="btn"] { font-weight: bold; }      /* class bắt đầu với "btn" */
input[class$="primary"] { color: blue; }        /* class kết thúc với "primary" */

/* Structural Pseudo-classes */
li:nth-child(odd) { background: #f0f0f0; }      /* Con lẻ */
li:nth-child(even) { background: #e0e0e0; }     /* Con chẵn */
li:nth-child(3n) { color: red; }                /* Mỗi 3 phần tử */
li:nth-child(3n+1) { color: blue; }             /* 1, 4, 7, 10... */
li:first-of-type { font-weight: bold; }         /* Đầu tiên cùng loại */
li:last-of-type { font-style: italic; }         /* Cuối cùng cùng loại */
li:only-child { text-decoration: underline; }   /* Con duy nhất */

/* CSS4 Selectors (mới) */
p:has(strong) { border-left: 4px solid blue; }  /* p chứa strong */
:is(h1, h2, h3):hover { color: red; }           /* Bất kỳ h1, h2, h3 nào */
:where(article, section) p { margin: 1rem 0; }  /* Tương tự :is nhưng specificity = 0 */

Best Practices

Phần này tổng hợp các best practices quan trọng nhất mà mọi Frontend Developer cần biết để viết code chất lượng cao, maintainable và scalable.

HTML Best Practices

  • Semantic HTML: Sử dụng các thẻ HTML5 semantic đúng mục đích
  • Accessibility: Luôn cung cấp alt text, ARIA labels, và keyboard navigation
  • Performance: Optimize images, lazy loading, và minify code
  • SEO: Proper meta tags, structured data, và semantic markup
  • Validation: Validate HTML code và fix all errors 

<!-- ✅ Good: Semantic và accessible -->
<article>
  <header>
    <h1>Tiêu đề bài viết</h1>
    <time datetime="2025-01-15">15 tháng 1, 2025</time>
  </header>
  <p>Nội dung bài viết...</p>
  <img src="image.jpg" alt="Mô tả chi tiết hình ảnh" loading="lazy">
</article>

<!-- ❌ Bad: Không semantic -->
<div>
  <div>Tiêu đề bài viết</div>
  <div>15 tháng 1, 2025</div>
  <div>Nội dung bài viết...</div>
  <img src="image.jpg">
</div>

CSS Best Practices

  • Mobile First: Thiết kế cho mobile trước, sau đó mở rộng lên desktop
  • BEM Methodology: Sử dụng naming convention rõ ràng
  • CSS Variables: Sử dụng custom properties cho maintainability
  • Performance: Avoid expensive properties và optimize selectors
  • Consistency: Maintain consistent spacing, colors, và typography 

/* ✅ Good: BEM naming, CSS variables, mobile first */
:root {
  --primary-color: #3498db;
  --spacing-unit: 1rem;
}

.card {
  background: white;
  border-radius: 8px;
  padding: var(--spacing-unit);
}

.card__title {
  font-size: 1.25rem;
  margin-bottom: calc(var(--spacing-unit) * 0.5);
}

.card__title--large {
  font-size: 1.5rem;
}

@media (min-width: 768px) {
  .card {
    padding: calc(var(--spacing-unit) * 2);
  }
}

/* ❌ Bad: Không có structure, hard-coded values */
.cardTitle {
  font-size: 20px;
  margin-bottom: 8px;
  color: #3498db;
}
.bigCardTitle {
  font-size: 24px;
  margin-bottom: 8px;
  color: #3498db;
}

Performance Optimization

  • Critical CSS: Inline critical CSS và lazy load non-critical
  • Image Optimization: WebP format, responsive images, lazy loading
  • CSS Optimization: Minify, remove unused CSS, use efficient selectors
  • Fonts: Font display strategies và preload important fonts
  • Animations: Use transform và opacity, avoid layout thrashing 

<!-- Critical CSS inline -->
<style>
  /* Chỉ CSS cho above-the-fold content */
  .header { /* styles */ }

Accessibility (A11y) Guidelines

  • Keyboard Navigation: Ensure all interactive elements accessible via keyboard
  • Color Contrast: Maintain WCAG AA contrast ratios
  • Screen Readers: Proper ARIA labels và semantic HTML
  • Focus Management: Visible focus indicators và logical tab order 

<!-- Accessible form -->
<form>
  <label for="email">Email Address *</label>
  <input type="email" 
         id="email" 
         name="email" 
         required 
         aria-required="true"
         aria-describedby="email-error">
  <div id="email-error" role="alert"></div>
</form>

<!-- Accessible navigation -->
<nav role="navigation" aria-label="Main menu">
  <ul>
    <li><a href="#home" aria-current="page">Home</a></li>
    <li><a href="#about">About</a></li>
  </ul>
</nav>

/* Accessible CSS */
.visually-hidden {
  position: absolute;
  width: 1px;
  height: 1px;
  overflow: hidden;
  clip: rect(0 0 0 0);
}

.focus-visible {
  outline: 2px solid #4285f4;
  outline-offset: 2px;
}
